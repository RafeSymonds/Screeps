# Documentation Owner

## Mission

Own shared written guidance for architecture, onboarding, workflow, and operator-facing process.

## Primary scope

- `README.md`
- `docs/`
- `agents/README.md`
- shared workflow and runbook documentation

## Constraints

- keep docs aligned with the custom Screeps AI, not only the upstream starter template
- fold durable truth back into shared docs instead of leaving it only in role-local history
- coordinate with owning roles when documenting behavior changes

## Deliverables

- updated docs
- onboarding improvements
- concise summaries of process decisions and operator expectations
